* **switch:**

```java
    //  switch语句具有穿透性
    //  也就是说整个switch遇到 break 才会退出，不然一路运行下去,也就是触法多个 case结构
    switch(a){
        // a这个变量必须是已经有确定的数值了
        case 1:
            // case 后面的必须是数值，即使是数值确定的变量都不行！
            // 数值后面的是`:`冒号
            执行语句1 ；
            break;
        case 10：
            执行语句2 ；
            break;  
        default :
            执行语句3 ；             
    }
    // default: 其实就是 case (除去其他case情况的所有情况):          
    // default: 没有`break`，所以可以放在任意位置，不影响switch的运行
    // default: 结构 可以不写,不写就不会有 case (除去其他case情况的所有情况):的判断了               
```

* **细致的讲解数组：**

* **方法**
    > **创建方法声明 :** 
```java
    返回值 xxxYyy(形参1 , 形参2){ 
        // `实参` 是传入的 数值；`形参`是方法中的变量  
        // 在函数中默认进行 `形参 = 实参` 的赋值操作    
        // 1. 传入基本数据类型的数据 ： 形参 = 数值 ，对形参操作并不会影响到实参   
        // 2. 传入引用数据类型的数据 ： 形参 = 地址 ，对形参操作会影响到数据本身，导致实参也受到影响  
// 所以我们要注意，方法对引用类型数据，也就是数组、实例的影响是直接改变它本身的 
        // 执行语句 ；
        return ;
        // 方法一定有 返回值 ，一定有return结构，二者相对存在 
        // return ; 是默认的，你不写返回值时，系统会帮你写
        // return ; 对应的返回值声明是 void ,就是什么都不返回
// 注意一点，return null ; 是返回 空指针 ，它对应的返回值声明是 引用数据类型 
    }
```          
    
* **方法重载 ：**  
一个类中方法是不能出现重复的  
当两个方法的**方法名**是一致的时候，无论返回值是不是同类型，这两个方法都是同一个    
但是，在此时，如果**形参**不同，这两个方法会被视为不同的方法，也就是**方法重载**   
而**形参不同的判断标准** 是依据形参类型的**种类和数量**，   
只要两个方法的**类型种类或者数量不一致**，这就是重载

* **作用域逻辑**  
**变量**在整个**类不同的位置中**，作用的范围是不同的
```java
class 类{
    int a = 10; //-> 全局变量,属于`class`类代码块，最外层的代码块
    // 类的成员属性
    方法1(){
        int b = 10； 
    }
    方法2(){
        if(){
            int c = 10;  // ->每次if被执行一次，c变量初始化一次
        }
        for(){
            int c = 10;  // -> 每次for循环体执行一次，c变量初始化一次
        }
// `if`和`for`两个代码块的作用域里，都定义 c变量 不产生冲突。
// 因为两个c只会在自己作用域中生效，不能彼此访问(平行关系)
    }
    --------------------------------
    作用域 的之间可以分为平行关系和包裹关系
    # 一个代码块，对应一个作用域。
      for结构、if结构、整个类class结构。具有{执行语句}这样的代码体，那他就是一个作用域

    # 内层的作用域可以访问外层的变量（包裹关系），
      也就是说外层的变量会作用到里层，里层的不能作用到外层

    # `方法1`和`方法2`(平行关系)
    是被`class类`作用域包裹的,不能在方法中重新定义a变量(包裹关系)
}
```
