**类和类之间的一种关系：类和对象的关系**

* 类，**作为模板，提供给 【对象】**   
【类】是完全服务于【对象】的  
【对象】的创建：  
` 类 对象名 = new 类() ;` --> 【对象】是引用类型数据   
可以简写成`new 类().var`  
对象名，是地址白变量：  
.toString( ) -> `包名称.类@地址`

之前说过【类】就是【文件】，所以我们创建的【对象】一般不在它的【类】里面的，也就是说**对象的创建和使用**是两个文件之间的事


***


* **一个【类】的基本结构如下**

```java
    package 包 ；
    import 【导入的类的全称】；

    public class XxxYyy{

        *public* int xxxYyy ;
        *public* int[ ] xxxYyy ;
    // 【成员属性】

        *public* 文件名XxxYyy ( ){

        }
    // 【构造器】,又称【构造方法】

        *public* void xxxYyy( ){
            实现功能的代码 ；
        }
    // 【成员方法】

        public void set成员属性(类型 arg) {
            this.成员属性 = arg ;
        }
        public 成员属性的类型 get成员属性( ) {
            return this.成员属性 ;
        }
    // 4.【getter/setter】方法 
    // 本质上就是【成员方法】， 功能是 返回/改变【成员属性】
    }
```

* **这些结构不用自己写！**  
    `insert(fn + delete触法) + alt`/`右键 + Generate`-->快速构建【类】的基本结构     

* **成员( 属性 | 方法 )**  
    1. **访问 ：**
    对象已经创建后，
    【对象名】.成员属性 -> 返回【成员属性】  
    【对象名】.成员方法( ) ; -> 调用【成员方法】

* **构造器（又称构造方法）**
    在【对象】创建的时候被调用，具有最高的调用优先级  
    一般用于【对象的成员属性】的赋值  
    * 【类】是模板：    

```java
    public class XxxYyy(形参1，形参2) {
        this.成员属性1 = 形参1;
        this.成员属性2 = 形参2;
    }
```

* 【对象】，又称【实例】，当 **【成员属性】是具体数值时**，对象就是一个基于【类】而创建的个体  
```java
    XxxYyy 对象名 = new XxxYyy(实参1，实参2) ;
```

* **【构造器的特性】：**    
    因为【构造器】是一种方法，所以可以**方法重载**  

    默认拥有【无参构造器】，  
    可是当你写入【有参构造器】时，不再提供如有需求，自己再写

> ## 类的成员变量 和 临时变量（局部创建/参数传入）

*  **变量优先级处理**  
    遵循【就近原则】
```java
    # 区分：
    <类外> - 【对象.成员】|【类.静态成员】|
    <类内> - 【this.实例成员】|【super.父类成员】|【类.静态成员】
    |【变量名】(*方法的定义的局部变量*)
    ---------------------
    # 什么时候用【成员属性】，什么时候用【临时变量】？

    1.当一个【变量】你想让它在整个对象中被访问，则是【全局变量】，
    用【成员属性】，这个值是共同作用的产物

    2.当一个【变量】你想让他只在一个【代码块】内部生效，则是【局部变量】，
    用【临时变量】接住一个值，这个值不会对外界产生影响（*除非是【返回值】*）

    3.方法的【参数】，本质上是接住【实参】，然后做【临时变量】接住它，
    只有当这个方法被嵌套在另一个方法中，作为【内部代码块】，
    用【参数】接住【外部代码块】的【临时变量】
```

**在类中重要的关键字**  
这些关键字在上一章的散点语法中没有出现，是因为在**一个类中**讨论他们是没有意义的，但是我们现在学习创建【类】、【对象】都将在**不同文件**中考虑，此时，他们至关重要！  

    * **【static】静态**  ，在【成员】的声明中使用
```java
    public class XxxYyy{
        public static int a;
        public static void xxxYyy( ) {

        }   
    //  加上static，就是【静态成员】，如果没有static就是【实例成员】
    //  静态成员是在【类】被创建的时候就存在了，而实例成员是【对象】被创建的时候才存在了
    }
```     
【static】给【成员】的**赋能作用：**  
【静态成员】不需要创建对象后，用`对象名.成员`的方法 访问或调用，他能直接用`类名.成员`的方法 访问或调用。我们也更提倡用这个方法  


在**类创建的时候**，`成员名` 就是【静态成员】 ，`this.成员`表示【实例成员】


【static】给【成员】的**限制作用：**  
是针对**类创建的时候**的限制，【静态成员方法】内只能访问【静态成员】

【静态成员属性】属于【类】，当它的数值被改变时，**会影响到【类】这一模板本身**   

* **【this】** 只服务于【实例成员】的关键字：  
它只会在**类中**被使用到，是在**类中**访问或调用【实例成员】的方法  
**其他类**中是`对象.成员`访问【实例成员】，**类创建的时候**则用`this.成员`访问


* **权限修饰符** ，在【成员】的声明中，或者在【类】的声明中使用，  
在一个类中权限修饰符不会产生作用的，他的作用是在**类与类**之间的。  
还有对【类】讨论权限修饰符暂不考虑，我们都用`public class XxxYyy{ }`

我们大部分情况下，只使用`private`和`public`这两种辨析成员【私有/非私有】  
**私有：** 只能在**类中**被访问  
**非私有** 则是整个**工程**中的所有类都能访问  
* 私有封装设计：  
【成员变量】全部`private`,   
统一用【getter/setter】方法，来设计可以被外界访问或改变的【成员变量】

**类和类之间的第二种关系： 继承 与 实现**

    
*  **子类【继承】父类：**   
    当有多个**相关的类**，把相同的【成员】抽出，构建【父类】  
    【子类】将继承所有【非私有成员】，继承与【访问的权限修饰符】无关，  
    **【子类】将继承所有【非私有成员】** 是继承的唯一规则
```java
    public class 子类 extends 父类{

        1.【成员： 成员属性、成员方法】
        // 当【成员】冲突（重名）时，符合<就近原则>，会访问子类的
// super关键字：
// 【super.成员】访问父类成员
// 跟【this】一样，只服务于【实例成员】- 父类的【实例成员】

        // 【成员方法】的重名，又称【方法重写】，这是一种 “适应继承多态特性”的好行为
        @Override
        public 方法声明与【父类】一致( ) {
            重写【方法体】 ;
        }
        // @override 起到(标明 + 检查重写语法)的作用

        2. 【构造器】
        // #子类【构造器】会默认“第一行”提供【super( )】方法
// super( )这个语法只能写在构造器中，且必须是第一行 
        // 因为生命周期原理：【父类成员】就位，子类才能继承，
        // 所以要求【父类构造器】必须优先【子类构造器】执行
        //  如果你写了【父类有参构造器】，此时，父类不会提供【父类无参构造器】，
        // 这时候，如果不走【super（有参）】，记得自己补一个【父类无参构造器】，防止【子类】的super( )语法引发报错    

        3.【getter/setter】
        // 继承父类时，是继承父类的【非私有成员】，但是【父类成员属性】又需要`private`封装保证，
        // 所以我们让子类直接继承 getter/setter 方法，保证子类可以访问和修改【父类成员属性】
    }
```

* **引言：**  
了解类和类之间父子关系，就是为接下来的**抽象类**和**接口**做准备  
这两中类都是**一种模板**，是对他们子类的**一种规范**，规范的方式就是用【抽象方法】   
他是由我们之前谈论的**类规范对象的创建**过渡到现在的**类规范类的创建**

* **抽象方法：**

```java
    public abstract class XxxYyy{
// 因为这个类有抽象方法，这个类必须是【抽象类】

    public abstract void man( ) ;
    // `abstract`关键字修饰
    // 没有【方法体】
    // 无法被调用，只能等【继承后重写】
    }
```
`abstract`与 `static`和`final` 的语义是冲突的，
抽象方法一般不用`private`,必须用`public`保证能被【子类】继承，实现【重写】

```java
    public class XxxYyy extends 【抽象类】{

        @Override
        public void 【抽象方法】() {
        // 一定要重写抽象方法
        }
    }
```

1. **有【抽象方法】就一定要是【抽象类】，但是【抽象类】可以不写【抽象方法】**  

2. 抽象方法对类的规范原理：   
提供**子类创建方法的声明**，强制了返回值、参数等要素。然后让子类给出具体的【方法体】   
**继承了【抽象类】，就要重写它所有的【抽象方法】 或者 【子类】也是【抽象类】**，【抽象类】继承【抽象类】时，不重写【抽象方法】，这个抽象方法也还是会被继承，所以最后那个**实现实例化的类**，一定是要**重写所有父辈类的未被重写的【抽象方法】**

3. **【抽象类】不能实例化，**，此时，【构造器】的作用就不是**创建对象时**起作用了，  
它的作用是**给子类继承时调用**

**引言：**  
【抽象类】只是在正常父类基础上添加【抽象方法】来提供子类**方法的模板**，他是在【类】的基础上额外增加一种**规范**   
而【接口】则是**专精的规范**，就是纯粹的规范【实现类】的多态行为   
当两个【类】的有方法相近时，我们大部分时候不是用【抽象类】来规范，除非他们本身**因为成员相近**就拥有相同的【父类】，  
我们一般都是创建【接口】，用【实现类】实现【接口】的方法为这些【实现类】提供模板，这就是**面向接口编程**



* **实现类【实现】接口** 

```java
    public interface IXxxYyy {
        // 接口不再是class 而是 interface ,文件名用I开头
        
        *public static final* XXX_YYY = 100;
        1.【静态常量】

        *public abstract* void xxxYyy(){

        }
        2.【抽象方法】
    //接口里面只放这两个东西
    // 这两个的【关键词】是默认提供的 
    }
```

【类】可以同时多继承多个【接口】，  
当一个【接口】同时实现多个【接口】,叫做**接口整合**  

```java
    class XxxYyy extends 【父类】 implements 【接口1】,【接口2】{

    }
```


**继承与多态**
java的类的继承链，都是**单链单继承**，一个【类】只能继承一个【类】，任何类都直接或者间接的继承 **【Object】始祖类** ，在java.lang 包中
* 所以始祖类的**public方法**所有类都会继承   
不【重写】的话用的就是用Object的方法

`toString()方法` – 设计对象的地址，返回String类型   
打印对象名时，就是默认调用 对象名.toString()

`equals()方法` – 等价于`==`的判断,返回boolean值   
基本数据类型之间判断**数值**   
引用数据类型之间判断**地址**

* **解耦合 思想：**
* 方法的创建中：  
```java
    public static void xxxYyy(父类 a) {
// 形参用【父类】，【子类】的对象都能作为 实参 
    }
```  
* 对象的创建中：   
```java
    父类 xxxYyy = new 子类（ ） ；  
```
这样创建对象的方式，你只需要**改动右半边的子类类型**就能创建新的对象。  
但是，这样的创建方式有个弊端：  
你的目的是创建一个子类的实例化对象，但此时这个创建方式会导致你的对象只能访问**从【父类】那里继承过来的成员**。子类自己创建的新成员不能被访问  
处理方式如下：  
```java
    子类 aaaBbb= (子类) xxxYyy；
    // 类型的强制转换，这样aaaBbb就能跟正常创建的对象一样，使用子类自己的成员   
```
但是在这个时候，如果创建的是【子类1】的对象，在强制转换时却是用的【子类2】，是会报错的   
**但是，这个报错并不会在编译时显示，**   
所以当你的方法传入一个实参时，他又是解耦合形式创建的，你可能会不小心传入**不是你想强制转换的子类**的实参，所以你要加个保险：
```java
    public void 方法(父类 a){
        if(a instanceof XxxYyy){
            XxxYyy b = (XxxYyy) a；
        }
    }    
```
